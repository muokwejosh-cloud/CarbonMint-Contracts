# ADR 0036: Define documentation standards

- Status: Accepted
- Deciders: arisu6804

## Context

The CarbonMint smart contract needs a clear, documented approach to "define documentation standards" so the codebase stays consistent and auditable.

## Decision

We define documentation standards as the standard for this contract, in line with Soroban best practices.

## Consequences

Improves clarity, testability, and maintainability, and gives future contributors a recorded rationale to build on.
