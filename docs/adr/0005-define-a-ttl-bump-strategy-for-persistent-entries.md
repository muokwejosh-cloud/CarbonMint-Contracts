# ADR 0005: Define a TTL bump strategy for persistent entries

- Status: Accepted
- Deciders: arisu6804

## Context

The CarbonMint smart contract needs a clear, documented approach to "define a ttl bump strategy for persistent entries" so the codebase stays consistent and auditable.

## Decision

We define a TTL bump strategy for persistent entries as the standard for this contract, in line with Soroban best practices.

## Consequences

Improves clarity, testability, and maintainability, and gives future contributors a recorded rationale to build on.
