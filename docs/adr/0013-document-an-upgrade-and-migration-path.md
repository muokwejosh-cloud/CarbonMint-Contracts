# ADR 0013: Document an upgrade and migration path

- Status: Accepted
- Deciders: arisu6804

## Context

The CarbonMint smart contract needs a clear, documented approach to "document an upgrade and migration path" so the codebase stays consistent and auditable.

## Decision

We document an upgrade and migration path as the standard for this contract, in line with Soroban best practices.

## Consequences

Improves clarity, testability, and maintainability, and gives future contributors a recorded rationale to build on.
