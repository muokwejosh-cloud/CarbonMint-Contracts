# ADR 0016: Standardize the unit test module layout

- Status: Accepted
- Deciders: arisu6804

## Context

The CarbonMint smart contract needs a clear, documented approach to "standardize the unit test module layout" so the codebase stays consistent and auditable.

## Decision

We standardize the unit test module layout as the standard for this contract, in line with Soroban best practices.

## Consequences

Improves clarity, testability, and maintainability, and gives future contributors a recorded rationale to build on.
