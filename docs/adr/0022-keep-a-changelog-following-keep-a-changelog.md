# ADR 0022: Keep a CHANGELOG following Keep a Changelog

- Status: Accepted
- Deciders: arisu6804

## Context

The CarbonMint smart contract needs a clear, documented approach to "keep a changelog following keep a changelog" so the codebase stays consistent and auditable.

## Decision

We keep a CHANGELOG following Keep a Changelog as the standard for this contract, in line with Soroban best practices.

## Consequences

Improves clarity, testability, and maintainability, and gives future contributors a recorded rationale to build on.
