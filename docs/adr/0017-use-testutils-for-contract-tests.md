# ADR 0017: Use testutils for contract tests

- Status: Accepted
- Deciders: arisu6804

## Context

The CarbonMint smart contract needs a clear, documented approach to "use testutils for contract tests" so the codebase stays consistent and auditable.

## Decision

We use testutils for contract tests as the standard for this contract, in line with Soroban best practices.

## Consequences

Improves clarity, testability, and maintainability, and gives future contributors a recorded rationale to build on.
